What it is

Not just a reverse proxy — a unified entry layer

Internet traffic enters on the left, passes through the LiteGate core engine for matching and governance in the middle, then reaches your backend infrastructure on the right. Site hosting, dynamic proxy, automatic certificates, service discovery and identity entry all live on one execution plane.

Capabilities

One gateway, nine jobs that used to need several components

Every item below is supported natively on the main path — no glue scripts.

Site & content hosting

Static sites, SPAs, Markdown and WebDAV: one site file hosts your frontend assets, no extra layer to bolt on.

Automatic HTTPS & certs

DNS-01, wildcards, auto-renewal and On-Demand TLS — manual issue, replace and reload folded back into the entry layer.

Dynamic DNS (DDNS)

A records sync automatically once a site is declared. Built-in Aliyun, Cloudflare, Huawei Cloud and Tencent Cloud — no pointing new domains by hand first.

Service discovery

Five providers — litemesh, litedeploy, consul, docker, discovery — add a label to go live; mixed sources merge automatically.

Cluster config center

In a multi-node cluster, site YAML and certificates live in Litemesh KV: write once, auto-sync and hot-reload across every node; certs are issued once under a distributed lock and shared cluster-wide — no per-node config to maintain.

Frontend releases

Zip the build to OSS and write one KV key; every node downloads, switches atomically and keeps previous versions for rollback.

MCP autopilot

Built-in MCP server with 20+ tools — AI assistants create sites, edit routes, read schemas, hot-reload and troubleshoot directly.

Identity & SaaS entry (IDS)

Remote auth, OIDC, tenant-context injection and sid routing — close to a real multi-tenant SaaS entry plane.

Security & observability

WAF, dynamic rate limiting, IP rules, CORS, Litemesh mTLS, plus access logs, Dashboard and Prometheus metrics.

Scenarios

Where it feels better than a plain reverse proxy

Not every project needs LiteGate, but these scenarios feel its value sooner.

Sites, docs & admin panels

Static sites and backend APIs behind one entry, with less glue and duplicate config.

Small products on private cloud or VPS

Single binary, Docker, automatic HTTPS and DNS-based certs fit lightweight, multi-domain reality.

Microservices

Services register to Litemesh and are consumed by LiteGate automatically; east-west calls run over mTLS zero-trust, and routing follows instances as they change.

Multi-tenant SaaS

Identity-driven routing, tenant-context injection and sid splitting — closer to what a SaaS entry actually needs.

Quick Start

Get your first response, then decide

The best first try: bring up a same-domain frontend + backend site in Lite mode.

# sites/hello.lite.yaml
localhost:
  respond: "Hello, LiteGate!"
$./litegate
$curl http://localhost/
# Hello, LiteGate!
$litegate -t
# config check passed

Bring a site up in a few lines of YAML

Just want to feel how LiteGate works? This is the shortest path. Same-domain frontend/backend and automatic HTTPS are ready by default.

Or let AI configure it

With the built-in MCP on, describe the scenario to AI — it reads the schema, generates config, saves it and verifies the entry is actually reachable.
