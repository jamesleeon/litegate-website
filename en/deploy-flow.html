First Go-Live

Consolidating a SaaS Launch: Two Distinct Mindsets

Take an "expense invoice" service. Traditional methods work, but scatter repetitive steps across releases. LiteGate streamlines these into a continuous workflow.

Traditional

Traditional Ingress Setup

Once features are complete, you must manually assemble peripheral ingress actions.

  • Manually prepare domains, VPS, web servers, and reverse proxies.
  • Point DNS first, then request HTTPS certificates.
  • Frontend static resources and backend APIs hosted in separate tools.
  • Rearrange upstreams, reload configuration, or sync nodes manually after backend scaling.
  • Upgrade frontends by manually distributing ZIP files and swapping directories across nodes.
LiteGate

The LiteGate Approach

Declare "I want to host a site," and let the entry layer absorb the remaining steps.

  • Prepare domains and nodes, and state your ingress intent clearly.
  • / routes to the frontend, /api to backends — consolidated at ingress.
  • Domain records and SSL certs provision automatically alongside site declarations.
  • Frontend version management runs on ZIP, OSS, and KV sync.
  • Backends connect to service discovery, absorbing instance churn natively.

Upgrade Flow

Ingress Lifecycle: Smooth Continuous Upgrades

LiteGate excels by eliminating repetitive operations during scaling, version shifts, and rollbacks.

Without LiteGate

Scaling operations get messy as load increases.

new backend instance
+ update upstream
+ reload gateway
+ sync static files
+ re-check https / domain
+ re-check new node state
With LiteGate

The entry layer absorbs changes, allowing teams to "declare new states" instead of fixing environments.

frontend:
+ upload zip to OSS
+ update one KV version key

backend:
+ start new instance
+ register to discovery

LiteGate:
+ auto sync
+ auto route
+ auto converge

Frontend Version Upgrades

Instead of rsyncing asset directories across machines, upload the ZIP and update a KV key — nodes self-converge.

Backend Node Scaling

With service discovery connected, new instances register and route automatically — no manual gateway reloads required.

New Node Expansion

New ingress nodes join seamlessly; frontend versions synchronize and backend traffic balances automatically.

Clean Rollbacks

With historical versions preserved, rollback becomes a simple version pointer shift; backends shift traffic across old and new instances.

LiteGate goes beyond simple packet proxying: it keeps deployment, scaling, and lifecycle workflows comfortable.
