可观测性与部署

把 LiteGate 稳妥地放到生产环境：用 Web Dashboard 实时看状态、用结构化访问日志审计流量、用 Prometheus 指标接监控告警，再用 systemd / Docker 做进程守护与开机自启。

Dashboard访问日志Prometheussystemd / Docker

1Web Dashboard

内置的可视化控制面板：在浏览器里看网关状态、在线改站点配置、查服务发现后端、看实时日志。出于安全默认关闭，需在 config.yaml 显式开启。

config.yaml
dashboard:
    enabled: true
    port: 9999
    username: "admin"
    password: "$2a$..."          # 必须是 Bcrypt 哈希，用 litegate -hash 生成
    token: "tk_xxx"              # 管理 API 的 X-API-Key
    allowed_origins: []          # 为空 = 仅同源；跨域访问时填具体来源
    require_secure_cookie: false # Dashboard 走 HTTPS 时设 true

生成密码哈希：

bash
./litegate -hash "your_strong_password"

实时指标

CPU、内存、Goroutines；RPS、累计请求、在线连接；所有域名证书过期倒计时。

站点管理

浏览器内直接编辑站点 .yaml 即时生效；每个站点的报错率与延迟。

服务发现

查看 Litemesh/Consul 在线服务，确认路由后端真实绑定的 IP 列表。

日志分析

实时访问日志流，关键错误定位与搜索。

生产安全三条

① 改掉默认用户名 admin；② 设置高强度 Bcrypt 密码；③ 限制 Dashboard 端口访问 IP（可在外层再套一层 ip_restriction，或只绑定内网）。

2访问日志

结构化审计日志，记录每个请求的路由 ID、最终上游地址、鉴权结果与各阶段耗时。可输出到 stdout，也可批量推送到外部收集端。

config.yaml
log:
    level: info
    format: json                 # 生产建议 json，便于采集
    access_log:
        enabled: true
        stdout: true             # 是否打到控制台
        routing_detail: false    # 是否记录详细选路决策（排障时开）
        webhook:
            enabled: false
            url: "http://mq-gateway:8080/access-log"
            batch_size: 100
            interval: "1s"
与日志告警配合

log.webhook（区别于 access_log.webhook）可把 error/fatal/panic 级别日志推送到 Slack / 飞书 / 钉钉。两者一个面向「全量访问审计」，一个面向「关键告警」。

3Prometheus 指标

暴露标准 Prometheus /metrics 端点，接入 Grafana 看板。

config.yaml
metrics:
    enabled: true
    port: 9090

或者启动时临时开启（不改配置）：

bash
./litegate -enable-metrics
prometheus.yml
scrape_configs:
  - job_name: litegate
    static_configs:
      - targets: ["127.0.0.1:9090"]

4pprof 性能剖析

需要排查 CPU / 内存 / Goroutine 问题时，开启 Go pprof 端点。

config.yaml
pprof:
    enabled: true
    port: 6060
bash
go tool pprof http://127.0.0.1:6060/debug/pprof/profile?seconds=30
仅限内网

pprof 与 metrics 端口都不应直接暴露到公网，务必绑定内网或加防火墙。

5systemd 部署（Linux）

把 LiteGate 装成 systemd 服务，实现开机自启与进程守护。

1) 安装二进制与目录

bash
sudo cp litegate /usr/local/bin/
sudo chmod +x /usr/local/bin/litegate
sudo mkdir -p /opt/litegate/{sites,streams,certs}
sudo cp config.yaml /opt/litegate/

2) 创建服务单元

/etc/systemd/system/litegate.service
[Unit]
Description=LiteGate - Cloud-Native API Gateway
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=/usr/local/bin/litegate -config /opt/litegate/config.yaml
WorkingDirectory=/opt/litegate
Restart=always
RestartSec=5
LimitNOFILE=65535

# 安全加固
NoNewPrivileges=true
ProtectSystem=strict
ReadWritePaths=/opt/litegate/certs /opt/litegate/sites

# 日志
StandardOutput=journal
StandardError=journal
SyslogIdentifier=litegate

[Install]
WantedBy=multi-user.target

3) 启用并启动

bash
sudo systemctl daemon-reload
sudo systemctl enable litegate
sudo systemctl start litegate
sudo systemctl status litegate
sudo journalctl -u litegate -f          # 跟踪日志
监听 80/443 的权限

不要用 root 跑。给二进制加特权端口能力即可：sudo setcap 'cap_net_bind_service=+ep' /usr/local/bin/litegate

更新无需改单元

升级时停服务、替换二进制、再启动即可。站点 sites/*.yaml 的改动不需要重启，LiteGate 会自动热加载。

6Docker 部署

bash
docker build -t litegate:latest .

docker run -d --name litegate \
  -p 80:80 -p 443:443 \
  -v $PWD/config.yaml:/app/config.yaml \
  -v $PWD/sites:/app/sites \
  -v $PWD/certs:/app/certs \
  --restart unless-stopped \
  litegate:latest
持久化证书

务必把 certs/ 挂成卷，否则容器重建会丢失已签发的证书、触发重复申请。多副本时改用共享存储（Litemesh/Consul KV）统一证书，详见 证书生命周期。

7生产上线清单

上线前

  • litegate -t 校验配置通过
  • log.format: json、级别设为 info
  • Dashboard 改默认用户名 + 强密码 + 限 IP
  • auto_cert.use_staging: false（正式证书）
  • real_ip.trusted_proxies 配置正确
  • LimitNOFILE / setcap 已就位

上线后

  • Prometheus 抓到 /metrics
  • 证书过期倒计时正常、自动续期生效
  • 访问日志正常落地 / 推送
  • metrics / pprof 端口未暴露公网
  • 多副本证书共享一致
  • 关键日志告警 webhook 已联通
零抖动热重载

新增域名、改灰度权重等都通过「离线构建新路由树 + 原子指针交换」生效，80/443 始终保持监听，正在传输的连接不受影响——不需要优雅重启。
