全局 config.yaml 参考

全局 config.yaml 配置的是网关级能力：监听端口、TLS 与自动证书、服务发现、缓存、Dashboard、日志与指标。它是可选的——不提供时网关用内置默认值启动；只有要开启上述能力时才需要它。站点路由不在这里，而在 sites/ 目录（见 YAML 与标签参考）。

网关级配置可选用 -init 生成模板
生成模板

运行 litegate -init 会生成一份带完整注释的 config.example.yaml，本页是对其各区块的逐项解读。改完用 litegate -t 校验。

1基础监听 http / real_ip

yaml
mode: observability        # performance（高性能少日志）/ observability（默认，含详细监控）

http:
    port: 80               # HTTP 监听端口（ACME HTTP 挑战也需要它）
    https_port: 443        # HTTPS 监听端口
    response_header_timeout: 30   # 等待上游响应头的超时（秒）

real_ip:
    trusted_proxies: ["127.0.0.1", "10.0.0.5"]   # 仅信任来自这些代理的 X-Forwarded-For
    selector_trusted_clients: ["127.0.0.1", "10.0.0.0/8"]  # 允许做东西向 X-Selector-Sid 物理选路的可信内网客户端
字段默认说明
modeobservabilityperformance 牺牲日志换吞吐；observability 保留完整监控与访问日志。
http.port80HTTP 监听端口。自动证书的 HTTP-01 挑战依赖它。
http.https_port443HTTPS 监听端口。证书就绪后自动启用。
http.response_header_timeout30等待上游返回响应头的最长秒数。
real_ip.trusted_proxies["127.0.0.1"]只有来自这些 IP/网段的请求，其 X-Forwarded-For 才被信任。
real_ip.selector_trusted_clients—允许携带 X-Selector-Sid 做物理分片选路的可信内网客户端。

2全局 IP 限制

服务器级防火墙，优先级最高，在站点匹配之前生效。

yaml
ip_restriction:
    # 白名单：非空时只允许这些 IP（慎用）
    # allow_ips: ["127.0.0.1", "192.168.0.0/16"]
    # 黑名单：拒绝指定 IP 访问任何站点
    deny_ips:
        # - "1.2.3.4"
注意优先级

allow_ips 一旦非空就变成「只允许名单内」，会拦掉其它所有来源。生产环境慎用白名单，更常用 deny_ips 黑名单。站点级也有各自的 ip_restriction，可叠加。

3TLS 与自动证书

两种路线二选一：静态证书（tls）或自动 ACME 证书（auto_cert，推荐）。原理详见 证书生命周期。

yaml
tls:
    enabled: false
    certs_dir: ./certs

auto_cert:
    enabled: true
    email: admin@example.com      # 证书到期通知邮箱
    certs_dir: ./certs
    renew_days: 10                # 到期前 10 天续期
    check_interval: 24            # 每 24 小时检查一次
    use_staging: false            # 测试时设 true，避免触发速率限制
    propagation_dns: ["223.5.5.5", "114.114.114.114"]

    # DNS Provider（阿里云/Cloudflare/腾讯/华为），用于 DNS-01 与泛域名
    dns_providers:
        # - name: "cf-global"
        #   type: "cloudflare"
        #   domains: ["*.example.com"]
        #   config: { api_token: "..." }

    # 按需 TLS（SaaS 模式）：为未知域名实时签发
    # on_demand:
    #    enabled: false
    #    ask: "http://your-backend/check-domain"
字段默认说明
auto_cert.enabledtrue启用自动 HTTPS。未在 dns_providers 匹配的域名默认走 HTTP-01。
auto_cert.emailadmin@litegate.comACME 账号邮箱，用于到期通知。
auto_cert.renew_days10证书到期前多少天触发续期。
auto_cert.use_stagingfalse测试时用 Let's Encrypt staging 环境，避免速率限制。
tls.enabledfalse使用本地静态证书（与 auto_cert 二选一）。

4注册中心与服务发现

consul / litemesh 用于网关自身的配置监听、证书存储与服务注册；service_discovery 是独立模块，定义后端服务从哪里发现。发现机制详见 服务发现与动态接入。

yaml
litemesh:
    enabled: true
    address: "192.168.1.100:8787"
    namespace: "default"
    roles:
        service_register: true     # 把 LiteGate 自身注册为服务

service_discovery:
    catalogs:
        - enabled: false
          provider: litemesh        # litemesh / litedeploy / docker / consul / discovery
          namespace: "default"
          default_domain: ""        # 服务名 -> service.example.com 的默认后缀
        - enabled: false
          provider: docker
          socket_path: "/var/run/docker.sock"
          network_mode: "internal"  # internal（容器 IP）/ external（宿主 IP:Port）
字段说明
litemesh.enabled / consul.enabled是否启用对应注册中心做配置监听 / 证书存储 / 自注册。
service_discovery.catalogs[].provider发现源：litemesh / litedeploy / docker / consul / discovery。
...default_domain给发现到的服务自动拼域名后缀（orders → orders.example.com）。
...network_mode（docker）internal 用容器 IP，external 用宿主机映射端口。
远程引导

也可以不写本地 config.yaml，用 litegate -consul-addr ... 或 -litemesh-addr ... 从 KV 拉取主配置启动；环境变量 LITEGATE_CONSUL_ADDR/TOKEN/KEY 同样支持。

5缓存与 Redis

IDS 会话、限流、负缓存等共享状态依赖 Redis；不启用时退化为本地内存（单机可用）。

yaml
cache:
    local_ttl: "60s"
    redis:
        enabled: false
        mode: "standalone"        # standalone / cluster
        address: "localhost:6379"
        db: 1
        pool_size: 20

redis:                            # 全局元数据存储
    enabled: false
    mode: "standalone"
    address: "localhost:6379"
    db: 0
字段默认说明
cache.redis.enabledfalse多节点共享会话/限流时启用；单机可不开。
*.modestandalonestandalone 或 cluster（集群用 addrs 列表）。
*.address / addrlocalhost:6379Redis 地址，addr 是 address 的别名。

6身份与认证

全局身份中心、OIDC 安全参数，以及可复用的 OIDC/OAuth2 与 IDS 提供商。IDS 开发详见 IDS 开发指南。

yaml
identity_service:
    endpoint: "http://localhost:8080"
    client_id: "gateway-001"
    client_secret: "your-secret"
    timeout: "5s"

oidc_security:
    cookie_secret: ""             # AES-256 cookie 加密密钥（配置则需 ≥ 32 字节）
    cookie_same_site: lax         # lax / strict / none
    max_concurrent_sessions: 10

# 可复用的认证提供商（被站点路由的 ids.provider 引用）
auth_providers:
    # google:
    #     type: oidc
    #     issuer: "https://accounts.google.com"
    #     client_id: "..."
cookie_secret 长度

oidc_security.cookie_secret 一旦设置就必须 ≥ 32 字节，否则启动校验直接失败。

7日志与可观测

日志、访问日志、指标、Dashboard。落地与排障细节见 可观测性与部署。

yaml
log:
    level: info                   # debug / info / warn / error
    format: console               # console / json
    access_log:
        enabled: true
        stdout: true              # 是否把访问日志打到控制台
        routing_detail: false     # 是否记录详细选路决策

metrics:
    enabled: false                # Prometheus 指标（也可用 -enable-metrics 开）
    port: 9090

pprof:
    enabled: false
    port: 6060

dashboard:
    enabled: false                # 出于安全默认关闭
    port: 9999
    username: "admin"
    password: ""                  # 用 litegate -hash 生成 Bcrypt 哈希
    token: "..."                  # 管理 API Token（X-API-Key）
    allowed_origins: []           # 为空 = 仅同源
    require_secure_cookie: false  # Dashboard 走 HTTPS 时设 true
字段默认说明
log.level / formatinfo / console级别与输出格式（生产建议 json）。
log.access_log.enabledtrue结构化访问审计日志开关。
metrics.enabled / portfalse / 9090暴露 Prometheus /metrics。
dashboard.enabled / portfalse / 9999Web 管理面板，默认关闭。
dashboard.password—必须是 Bcrypt 哈希，用 litegate -hash "pw" 生成。

8其他与 MCP

yaml
sites_dir: ./sites            # 站点配置目录
streams_dir: ./streams        # L4 TCP/UDP 流配置目录
disable_trace_id: false       # 全局关闭 Trace ID 注入

mcp:
    enabled: false            # 内建 MCP 服务器，供 AI 助手接管运维
    port: 8080
    # token: "your-auth-token"
字段默认说明
sites_dir./sitesL7 站点 YAML 目录，支持热加载。
streams_dir./streamsL4 TCP/UDP 流代理配置目录。
disable_trace_idfalse全局禁用 X-Trace-Id 注入。
mcp.enabled / portfalse / 8080MCP 服务器，详见 MCP / AI 自动驾驶。

9完整示例

带完整注释的标准模板由 litegate -init 生成。最小可用的全局配置可以只有几行：

config.yaml
mode: observability
http:
    port: 80
    https_port: 443
auto_cert:
    enabled: true
    email: admin@example.com
log:
    level: info
    format: json
sites_dir: ./sites
改完先自检

修改后用 litegate -t 校验配置正确性再上线；站点 sites/*.yaml 的改动不需要重启，会自动热加载。
