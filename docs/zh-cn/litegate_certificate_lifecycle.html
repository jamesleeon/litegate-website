让用户忘记证书文件，把 HTTPS 变成网关的生命周期能力

LiteGate 的目标不是让用户绕过证书规则，而是让用户不再手动申请、拷贝、续期、reload 证书。用户声明域名、DNS provider 或授权策略，LiteGate 在运行时完成证书查找、签发、存储、续期和热加载。

自动 HTTPSOn-Demand TLS多节点存储ACME

1为什么这值得单独写一篇

在传统反向代理里，证书通常是一个外部前置动作：先用 certbot、acme.sh 或运维脚本拿到 .pem/.key，再把文件放到指定目录，最后 reload 代理进程。

LiteGate 把这件事收进入口层：TLS 握手通过 SNI 查找证书，缺失时可以触发按需签发；签发结果写入权威存储并同步到本地；续期、证书链校验、多节点锁、SaaS 域名授权都在同一个证书管理器里完成。

准确的产品表达

不是“用户不需要域名所有权”，而是“用户不需要管理证书文件”。域名解析、DNS API 凭证、HTTP-01 端口或 SaaS 授权策略仍然是安全边界的一部分。

2三条证书路径

已有证书加载

TLSManager 扫描证书目录，读取 .pem/.crt 和 .key，按 CommonName 与 SAN 建立内存索引，支持通配符证书。

启动时确保证书

配置了静态 dns_providers 时，AutoCertManager 会在启动和周期检查中确保证书存在、SAN 正确、证书链有效，并在需要时申请或续期。

首次访问按需签发

开启 on_demand 后，如果握手时没有命中的证书，LiteGate 会先做域名授权，再通过 ACME 为这个 SNI 域名签发证书。

3一次 HTTPS 握手内部发生了什么

  1. 读取 SNI
    客户端发起 TLS 握手，LiteGate 从 ClientHello 读取 ServerName。
  2. 规范化域名
    域名转小写、做 IDNA 规范化，并拒绝非法 hostname。
  3. 查找内存证书
    先精确匹配，再按一层通配符规则匹配 *.example.com。
  4. 按需兜底
    仍未命中时调用 On-Demand handler，授权通过后申请证书并放入内存缓存。
flow
ClientHello(SNI)
  -> TLSManager.GetCertificate
  -> exact / wildcard memory match
  -> OnDemandHandler
  -> ACME obtain
  -> storage + local disk
  -> memory cache
  -> handshake continues

4ACME 校验如何处理

HTTP-01

LiteGate 在 HTTP 路由最前面拦截 /.well-known/acme-challenge/，直接返回 ACME token。使用这条路径时，公网 80 端口需要能到达 LiteGate。

DNS-01

配置 DNS provider 后，LiteGate 可以通过 DNS API 写入校验记录，适合通配符证书和 80 端口不可开放的环境。

如果启用了 DNS 更新，LiteGate 还可以在申请前同步域名解析，避免“服务已经注册，但 DNS 还没指向网关”的上线断点。

5多节点时证书不会各自为政

证书可以落在本地磁盘，也可以使用 Consul 或 Litemesh 作为权威存储。权威存储成功后，LiteGate 会把证书镜像到本地证书目录，供 TLSManager 快速加载。

能力作用
分布式锁多个 LiteGate 节点同时遇到同一域名时，只有一个节点负责向 CA 申请，其他节点等待并读取结果。
pending 状态Litemesh 存储可标记正在申请的域名，降低脑裂或锁竞争导致的重复签发风险。
确定性延迟申请和续期前加入抖动，避免集群在同一毫秒集中冲击 CA 或 DNS provider。
本地镜像权威存储用于一致性，本地磁盘用于快速加载和文件变化监听。

6续期与热加载

LiteGate 周期性检查静态证书和动态域名证书。检查内容包括证书链完整性、过期时间、SAN 是否符合预期，以及 On-Demand 域名是否仍然被授权。

签发或续期成功后，证书写入权威存储并同步到本地目录，然后触发 TLSManager 重新加载。外部文件变化也会被证书目录 watcher 捕获并做防抖 reload。

用户真正少做了什么

不需要手动复制 PEM/KEY，不需要写 reload 脚本，不需要在每个节点重复申请，不需要单独维护证书过期巡检。

7安全边界仍然清晰

域名授权

On-Demand 域名必须来自已定义站点、Litemesh KV 或 SaaS Ask URL。全局通配站点不会直接授权任意域名。

Ask URL

Ask 默认要求 HTTPS。需要内网 HTTP 时必须显式开启 allow_insecure_ask；也可以配置 ask_ca 校验私有 CA。

账户密钥

storage_secret 可用于加密 ACME account key，推荐在共享存储场景启用，长度至少 32 字节。

9用户最少需要配置什么

公网固定域名

auto_cert:
  enabled: true
  email: ops@example.com
  renew_days: 30
  dns_providers:
    - name: cloudflare
      enabled: true
      domains: ["app.example.com"]
      provider_config:
        api_token: "${CF_API_TOKEN}"

SaaS 按需域名

auto_cert:
  enabled: true
  email: ops@example.com
  on_demand:
    enabled: true
    ask: "https://control.example.com/tls/ask"
    saas_template_site: "tenant-template"
  storage_secret: "${AUTOCERT_STORAGE_SECRET}"
上线前检查

DNS API 凭证、80/443 端口、CA rate limit、Ask 服务可用性、共享存储权限和 storage_secret 管理方式，仍然需要作为生产发布清单的一部分。
