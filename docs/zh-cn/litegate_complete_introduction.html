English version

一套入口模型,统一站点、API、服务发现、证书、身份决策与 mTLS

LiteGate 不只是反向代理。它把 Caddy 式的站点接入体验和 Traefik 式的动态后端能力收进同一个网关:托管静态站点、代理 API、发现动态服务、自动签发证书、执行 IDS 身份决策,并通过 Litemesh mTLS 安全访问后端——全部运行在一个零锁、可纳秒级热重载的数据面之上。

1从最小可用入口开始

LiteGate 可以先用静态 YAML 跑起来,再逐步切换到服务发现、自动证书、IDS 和 mTLS。左边是极简模式(.lite.yaml),右边是等价的标准模式——同一套路由模型。

lite
app.example.com:
  /api:
    proxy: 127.0.0.1:8080
  /:
    root: ./dist
    spa: true
yaml
domain: app.example.com
routes:
  - name: api
    match: { path_prefix: /api }
    action:
      type: proxy
      upstream: ["127.0.0.1:8080"]
  - name: frontend
    match: { path_prefix: / }
    action:
      type: serve
      root: ./dist
      spa: true
同一套路由模型

后续可以无缝切到 Litemesh、LiteDeploy、Docker、Consul 或 Discovery 发现源,也可以挂上 IDS 与 mTLS,而不需要重写配置心智模型。

2一个请求在网关内部怎么流转

LiteGate 概念上分 L1–L7,但物理执行顺序是经过反转优化的:网络防御与限流前置、安全与身份注入中置、负载均衡与链路治理后置。这样即使后端鉴权服务异常,网关也不会被无效流量打穿。

L1 · 入口防御

Router.ServeHTTP

全局 IP 过滤(Fast-Fail)、剥离客户端伪造的 X-Tenant-ID/X-SID 等身份头、路径规范化、拦截 ACME 挑战与 /_litegate/ 系统路径。

L2 · 路由匹配

Site Match

在只读 Trie 树中按 Host/Path 快速命中站点 [id] 与具体 Route,并把 MatchedPrefix 注入执行上下文。未命中返回 404。

L3 · 业务容器

CoreHandler

生成符合 W3C 规范的 X-Trace-Id、挂载 Panic 恢复、初始化结构化日志容器。单个中间件 panic 也只返回 500,进程不挂。

L4 · 中间件链

CORS → IP → WAF → RateLimit → Auth → Compress

限流在认证之前执行,保护下游 IDS/Redis 不被高频无效请求压垮。Auth 成功后把租户元数据(X-Tenant-ID、X-SID、X-DB-Name)注入 Header 与 Context。

L5/L6 · 代理执行

Proxy Executor

路径改写(Strip/Prepend)、灰度抽签、两阶段实例过滤(强隔离 selector + 软偏好 meta)、LB 调度(P2C/RR/Weighted/IpHash)、超时/重试/熔断治理包裹。

后端

物理发包并回写响应

访问命中的健康实例;若声明了 mTLS,则把链路升级为 Litemesh mTLS。

L7 · 可观测性

Observability (defer)

请求结束聚合整条流水线:路由 ID、最终上游地址、鉴权结果、各阶段耗时,输出结构化 JSON 日志。

3架构一览:控制面、数据面与原子热重载

控制平面

loader 把本地文件(fsnotify)、Consul Catalog、Litemesh 成员状态、外部 API 注册中心和 MCP 推送,统一规范化为 ActiveSite / ActiveStream。配置流与流量流彻底解耦。

数据平面

路由器只持有一个只读路由树指针(atomic.Value),无全局 Mutex。所有上游编译并缓存进 PoolManager,数据面以"零锁、零分配"运行,做到极高吞吐。

原子热重载

新增域名或改灰度权重时,后台离线构建全新 Trie 树,再用原子指针交换瞬间切换。构建失败则指针不切换,旧树继续服务,达到企业级 Fail-Safe。

无抖动

不需要 fork 进程,也不需要优雅关闭端口。80/443 从始至终保持监听,旧树在处理完遗留请求后由 GC 异步回收,正在传输的连接不受影响。

5为什么它又快又省

LiteGate 能在笔记本级配置上跑出 ~5.5w RPS、常驻内存 ~50MB,核心在于对代理连接池(PoolManager)的极致调优,以及数据面的零锁/零分配设计。

调优点做法收益
单 Host 空闲长连接MaxIdleConnsPerHost: 4096(传统网关常为 2~100)高并发下直接复用长连接,把 TLS/TCP 握手开销摊薄为零,避免耗尽临时端口
零锁数据面只读路由树 atomic.Value,无全局 Mutex高并发路由匹配无锁竞争
零分配调度上游预编译缓存进 PoolManager请求流经数据面不再反射或重分配内存
动态传输清理后台每 5 分钟回收空闲连接实例缩容后平滑归还内存,常驻保持低水位
