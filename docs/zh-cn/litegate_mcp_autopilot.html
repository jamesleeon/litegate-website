内建 MCP 服务器:让 AI 助手直接接管网关运维

LiteGate 内建 Model Context Protocol(MCP)服务器,天然支持与 AI Agent 无缝集成。用 Cline、Cursor、Roo-Code 等助手维护网关时,AI 可直接调用内置工具链:获取环境、校验配置、热重载、读取实时报错与证书状态——实现"自动驾驶"级运维。用户只要配好 MCP、给出业务意图,通常不再需要额外提供 LiteGate 文档。

1启用与连接

在网关 config.yaml 勾选 mcp 配置块,再在 AI 客户端的 mcp_settings.json 里指向 SSE 端点即可。

config.yaml
mcp:
  enabled: true
  port: 8080   # MCP SSE 服务监听端口
mcp_settings.json
{
  "mcpServers": {
    "litegate-autopilot": {
      "type": "sse",
      "url": "http://127.0.0.1:8080/mcp/sse"
    }
  }
}
客户端配置位置

以 Cline / Roo-Code 为例,通常位于 %APPDATA%\Code\User\globalStorage\saoudrizwan.claude-dev\settings\mcp_settings.json。连接成功后,AI 即可通过工具集帮你建站、热加载路由或排查故障。

3典型闭环

AI 通常按下面的链路把一类需求跑完整,从盘点到验证形成闭环。

① 创建反向代理网站

detect_environment→test_upstream→create_proxy_site→preview_config_change→save_site_config→verify_site→get_recent_errors

② 将域名绑定到服务发现中的服务

list_services→get_service_detail→bind_domain_to_service→preview_config_change→save_site_config→verify_site

③ 创建 TCP/UDP Stream

test_upstream→create_stream_proxy→preview_config_change→save_stream_config→verify_stream→get_recent_errors

④ 动态前端 ZIP 自动同步与升级(Serve KV)

detect_environment→get_gateway_config→set_kv→verify_site→get_recent_errors

向 KV(如 litegate/config/front/main)写入最新前端 ZIP 版本号、URL 与 SHA256,网关自动热切部署,再发起真实请求验证。

4当前覆盖度判断

如果不把认证 / OIDC 作为前提,当前 MCP 已足够让 AI 在没有外部文档的情况下完成大多数常见建站与日常管理。

为什么可以"不给文档"

MCP 本身已暴露 schema 与配置说明、当前运行状态与现有配置,并提供校验、保存、回滚、验证、排障能力。因此外部文档不再是常见建站流程的必需输入,而更像补充材料。

5仍然建议继续增强的点

  • preview_config_change 的影响分析可以更细,明确受影响的站点、端口、监听器。
  • verify_site 可以进一步返回命中的 route、最终 upstream 等更强断言信息。
  • verify_stream 可以进一步确认监听者是否为 LiteGate 本进程。
  • 回滚链路可以进一步强化备份失败时的错误处理和状态说明。
